# Оцінювання ретросинтезу

> Цей файл фіксує ручні, якісні прогони (по 0-10 балів на молекулу) на невеликому
> наборі цільових продуктів. Кількісну частину Розділу 4 ПЗ (Top-1/3/5 exact match,
> Structure Success Rate на повному test split USPTO-50K, Zero-shot проти RAG+CoT)
> тепер дає автоматизований `scripts/evaluate_retrosynthesis.py`, який прогонить
> будь-який зареєстрований `LLMProvider` через той самий одно-кроковий CoT-конвеєр
> і друкує markdown-таблицю у форматі Табл. 4.1. Ручні прогони нижче лишаються як
> якісний, пояснювальний журнал — вони перевіряють, чи міркування має сенс, а не
> лише чи збігається SMILES.

Для кожного еталонного продукту запускаємо 5 режимів:

- A. Groq без RAG: Groq API, модель `llama-3.3-70b-versatile`
- B. Гібридний RAG: Groq API, модель `llama-3.3-70b-versatile`; схожість продукту + схожість реакційної трансформації + схожість класу реакції
- C. ReactionT5v2 без RAG: `sagawa/ReactionT5v2-retrosynthesis-USPTO_50k`
- D. ChemLLM-20B-Chat-SFT GGUF без RAG: `mradermacher/ChemLLM-20B-Chat-SFT-i1-GGUF`, файл `ChemLLM-20B-Chat-SFT.i1-Q4_K_M.gguf`
- F. Two-stage fine-tuned Qwen2.5-7B LoRA без RAG: reactant/class adapter `oleh13/retro-reactants-qwen25-7b-lora` + condition adapter `oleh13/retro-conditions-qwen25-7b-lora`
- G. ReactionT5v2 + Qwen-умови без RAG: прекурсори `sagawa/ReactionT5v2-retrosynthesis`, умови LoRA `oleh13/retro-conditions-qwen25-7b-lora` (клас реакції невідомий, плейсхолдер `unclassified`)

У режимах A і B через Groq API використовувалась модель `llama-3.3-70b-versatile`.

Конфігурація режиму D:

```python
llm = Llama.from_pretrained(
    repo_id="mradermacher/ChemLLM-20B-Chat-SFT-i1-GGUF",
    filename="ChemLLM-20B-Chat-SFT.i1-Q4_K_M.gguf",
    n_ctx=2048,
    n_gpu_layers=25,
)
```

Конфігурація режиму F:

- Base model: `Qwen/Qwen2.5-7B-Instruct`
- Reactant/class LoRA: `oleh13/retro-reactants-qwen25-7b-lora`
- Condition LoRA: `oleh13/retro-conditions-qwen25-7b-lora`
- Inference pipeline: `product -> reactants/class -> conditions`, greedy decoding, app-compatible route JSON conversion
- Source notebook: `fine-tune/v2/res/06_agent_inference_demo.ipynb`

Конфігурація режиму G:

- Reactants model: `sagawa/ReactionT5v2-retrosynthesis` (без префіксу задачі, beam search `num_beams=5`, без класу реакції)
- Condition LoRA: `oleh13/retro-conditions-qwen25-7b-lora` на базовій `Qwen/Qwen2.5-7B-Instruct`
- Клас реакції: ReactionT5 не передбачає клас реакції, тому в запит до умов передається плейсхолдер `unclassified`
- Inference pipeline: `product -> reactants (ReactionT5, beam search) -> conditions (Qwen LoRA)`; конвертується у app-compatible route JSON лише перший (найкращий за beam-score) набір прекурсорів
- Source notebook: `fine-tune/v2/res/08_reactiont5_conditions_agent_demo.ipynb`

Оцінюємо найкращий запропонований одностадійний маршрут. Максимум: 10 балів на молекулу.

## Критерії

### 1. Збіг прекурсорів, 0-8 балів

Критерій одночасно оцінює точність підібраних прекурсорів і коректність визначеного класу реакції для еталонного продукту:

| Прекурсори | Клас реакції неправильний | Клас реакції приблизно правильний, надто загальний | Клас реакції правильний |
|---|---:|---:|---:|
| 0 = неправильні прекурсори | 0 | 1 | 2 |
| 1 = частково правильна ідея, ключовий прекурсор неправильний або відсутній | 2 | 3 | 4 |
| 2 = правильний тип прекурсорів, практичний набір визначено не повністю | 4 | 5 | 6 |
| 3 = очікувані або хімічно еквівалентні прекурсори, що дають той самий одностадійний тип трансформації | 6 | 7 | 8 |

### 2. Якість метаданих, 0-2 бали

Єдиний критерій, що оцінює умови реакції: розчинник, час, вихід, температуру, очищення та виділення і будь-які інші подібні чинники.

- 0 = вигадані точні умови або хімічно неприйнятні реагенти
- 1 = умови правдоподібні, але неповні або без чіткого джерела
- 2 = реагенти й умови правдоподібні та позначені як RETRIEVED / ESTIMATED / UNKNOWN

Хімічно еквівалентні прекурсори та практичні варіанти тієї самої одностадійної трансформації можуть отримати максимальний бал. Альтернативний маршрут іншого класу оцінюється за фактичною відповідністю еталонній реакції, навіть якщо він загалом хімічно можливий.

### Інтегрована точність

- Інтегрована точність розраховується із сумарної оцінки: сума балів / 10 * 100.
- Це не дублює точність окремим критерієм, а використовує всю рубрику як єдине джерело оцінки.
